## 这个功能是什么

为角色扮演网站实现一套完整的角色卡面系统。用户可以给自己的角色生成一张精美卡片，用来观赏、导出、分享给朋友，朋友点链接就能直接导入这个角色开始聊天。

---

## 三种卡的分工

系统里同时存在三种不同用途的卡，各司其职：

**① 全息视差卡（角色列表用）**
放在侧边栏的角色列表里。鼠标悬停时卡片3D倾斜，彩虹棱镜光效跟随鼠标实时变化。每个角色都显示为这种卡，用于日常浏览切换角色。实现细节见 CARD-IDE-SKILL-v2.md 第一节。

**② 封印揭示动画（新角色导入时用）**
当用户通过分享链接导入一个新角色时，触发一次封印揭示动画：先显示魔法球封印面，然后光爆炸开，角色卡浮现出来。同一个角色只触发一次，之后正常显示。实现细节见 CARD-IDE-SKILL-v2.md 第二节。

**③ 风格展示卡（导出分享用）**
用户专门为角色生成的精美卡面，用于导出和分享。支持 11 种预设风格 + AI 自由创作风格 + 玩家手动自定义，详见下方功能说明。

---

## 风格展示卡的完整功能

### 入口位置
在角色设置面板的「角色」Tab 里，加一个「生成角色卡」按钮。点击后弹出卡面工作台弹窗。

---

### 卡面工作台弹窗（核心界面）

弹窗分左右两栏：
- **左栏**：卡片实时预览，始终显示当前参数渲染的卡面效果
- **右栏**：操作区，包含以下四个 Tab

---

#### Tab 1：AI 生成

点击「AI 生成卡面」按钮，调用网站 AI（使用 CARD-WEB-SKILL-v2.md 作为系统提示），传入当前角色的名字、性格、背景、说话风格，AI 返回卡面参数 JSON，左栏立即渲染预览。

生成后显示三个操作：
- **「重新生成」**：AI 换一种不同风格再生成一次
- **「提意见」**：用户输入文字（如"颜色太暗了"、"换国风"、"标题重写一个"），AI 根据意见调整，左栏实时更新
- **「满意，使用这个」**：确认当前卡面，进入导出流程

提意见的输入框要一直显示，用户可以反复提，AI 反复改，直到满意为止。

---

#### Tab 2：预设风格

显示 11 种预设风格的缩略预览卡，用户点击任意一种，左栏立即切换到该风格渲染当前角色信息。

11 种风格：国风水墨、甜心恋爱、暗黑哥特、赛博朋克、冒险热血、治愈日常、故障艺术、老式胶片、极简主义、液态极光、手账剪贴。

切换风格后，文案由 AI 自动重新生成（匹配该风格的语气），不需要用户手动改。

---

#### Tab 3：自定义

用户手动调整卡面参数，所有修改实时反映在左栏预览里。

提供以下调整项：
- 背景色（颜色选择器）
- 强调色（颜色选择器）
- 标签文字（输入框）
- 标题文字（输入框）
- 描述文字（输入框，限35字）
- 边框样式（下拉：细线/双线/虚线/发光/无）
- 字体风格（下拉：衬线体/等线体/手写体/粗黑体）
- 是否显示光晕（开关）
- 数值展示（可添加/删除，每条包含标签和值）

底部加一个「让 AI 优化文案」按钮，保持当前视觉参数不变，只让 AI 重写 tag/title/desc 三个文字字段。

---

#### Tab 4：导出与分享

显示当前卡面的最终预览（大图）。

提供两种导出方式：

**方式一：分享链接**
点击「生成分享链接」，把完整的 CardData（含角色人设数据）base64 编码后拼接到 URL hash 里，格式为：
`当前页面URL#import=base64编码内容`

链接生成后显示在输入框里，旁边有「一键复制」按钮。
同时显示提示："朋友打开这个链接，就能直接导入你的角色开始聊天"

**方式二：下载 JSON**
点击「下载角色卡文件」，下载 CardData 的 JSON 文件，文件名为 `角色名_card.json`，可作为备份或离线分享。

---

### 导入流程

**链接导入：**
页面加载时检测 URL hash，如果包含 `#import=`，解析 base64 数据，弹出导入确认弹窗：
- 左侧显示卡片预览（触发封印揭示动画，自动播放）
- 右侧显示角色名、描述、作者留言（如果有）
- 底部两个按钮：「导入角色」和「取消」
- 点击导入后：角色加入角色列表，自动切换到该角色，清除 URL hash

**文件导入：**
在侧边栏底部加一个「导入角色卡」按钮，点击后弹出文件选择框，选择 JSON 文件，解析后走同样的导入确认弹窗流程。

---

## 数据结构

CardData 的完整结构见 CARD-IDE-SKILL-v2.md 第四节，必须严格按照文档里的 TypeScript 接口定义实现，不要自行修改字段。

重点注意：`roleData` 字段包含完整的角色人设数据，导入后可以直接开始聊天，不需要用户重新填写人设。

---

## 技术要求

- 基于现有项目的 Vue 3 + Composition API 实现，不引入新的状态管理库
- 卡面预览用纯 CSS + 少量 JS 实现，不依赖 Canvas 或第三方图表库
- 所有弹窗用现有项目的弹窗组件风格保持一致
- 新增组件放在 `src/components/` 目录下，命名：
  - `CardWorkshop.vue` — 卡面工作台主弹窗
  - `CardPreview.vue` — 卡片预览组件（可复用）
  - `CardImportModal.vue` — 导入确认弹窗
- 新增 composable 放在 `src/composables/` 目录下：
  - `useCardSystem.js` — 卡面生成、导入导出的核心逻辑

## 实现顺序建议

1. 先实现 `CardData` 数据结构和 `useCardSystem.js` 的基础逻辑（生成链接、解析链接、导入导出）
2. 实现 `CardPreview.vue`，把 11 种预设风格的渲染都写进去，以及 custom 风格的动态渲染
3. 实现 `CardWorkshop.vue` 弹窗，先把 Tab 2（预设风格）做出来验证渲染效果
4. 接入 AI 生成（Tab 1），把 CARD-WEB-SKILL-v2.md 作为系统提示传给 AI
5. 实现 Tab 3 自定义调整
6. 实现 Tab 4 导出分享
7. 实现导入流程（链接导入 + 文件导入 + 封印揭示动画）
8. 最后把全息视差卡接入侧边栏角色列表

每一步做完都要在浏览器里验证效果再进行下一步。
