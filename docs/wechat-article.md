# 我花了几个月，给自己做了一个沉浸式 AI 角色扮演网站

**你和 AI 聊天，到底是在聊什么？**

很多人用 AI 聊天，但聊了两轮就没感觉了——要么 AI 记不住你说过的话，要么角色扮演到一半突然"出戏"，要么只能单调地一问一答。

我也有同样的困惑，所以我自己做了一个。

今天分享给你。

## 🎭 这是什么？

**MyAI RolePlay** —— 一个运行在浏览器里的 AI 角色扮演平台。

你可以在这里：
- 创建任意 AI 角色，给她/他设定性格、说话方式、外貌、秘密
- 跟 AI 深度对话，她会记住你们经历的一切
- 进入小说世界，成为书中人物亲历故事
- 开一场多人群戏，让几个 AI 角色同时陪你演剧情

所有数据存在你自己浏览器里，不联网，不收集，没有服务器能看到你的对话。

**网址：myai-app.pages.dev**（手机也能用，可以添加到主屏幕）

---

## ⚙️ 使用前要做一件事：配置 API Key

这个网站本身完全免费，但跟 AI 对话需要你提供自己的 API Key。

> **API Key 是什么？** 就是你调用 AI 服务的"通行证"，费用自己承担，按量计费，一般聊很久才花几块钱。

下面手把手教你注册。

---

### 推荐平台：DeepSeek（最简单）

DeepSeek 的 deepseek-chat 模型擅长中文创作，价格极低（约 ¥0.14/百万输入token），**新用户送 10 元体验金**。

**注册步骤：**

**第一步** → 手机/浏览器打开：`platform.deepseek.com`

**第二步** → 右上角点「注册」，用手机号或邮箱注册

**第三步** → 登录后，点左侧菜单「API Keys」→「创建 API Key」

【图片位置：DeepSeek平台 API Keys 页面截图】

**第四步** → 给这个 Key 随便起个名字（比如"MyAI"），点创建

**第五步** → 复制弹窗里那一串以 `sk-` 开头的字符，**这就是你的 API Key，只显示一次，记得保存！**

---

### 填入网站的步骤：

**第一步** → 打开网站后，点右上角 ⚙️ 图标（设置）

**第二步** → 选第一个「模型」Tab

**第三步** → 在「平台预设」里点一下「DeepSeek 官方」，Base URL 会自动填好

**第四步** → 在「API Key」框里粘贴你刚才复制的 `sk-` 开头的字符串

**第五步** → 模型选 `deepseek-chat`（已默认），点「保存」

【图片位置：网站设置页面截图，显示模型Tab】

✅ 配置完成！现在可以开始创建角色了。

## 🎮 怎么玩？

### 创建你的第一个角色

点首页「+ 新角色」按钮，然后：

1. **起个名字**，上传一张头像（也可以用默认的）
2. **填写人设**——性格、说话风格、跟你的关系，写多写少都行
3. 点「开始对话」

【图片位置：角色创建设置页面截图】

> 💡 人设不用写很完整，你可以边聊边修改。AI 会根据你写的内容来扮演角色。

---

### 几个你可能很快就会用到的功能

**「剧情分支」** ——每条 AI 回复左下角有个分叉箭头。点它，可以在这一刻之后重新开始一条不同的剧情线。探索不同走向，两条线互不干扰。

【图片位置：聊天界面分支按钮截图】

**「世界书」** —— 在角色设定里，可以给角色添加世界书（比如"修仙世界的门派体系""主角的家族历史"）。聊天时只要对话里提到相关词，AI 就会自动带上这些设定背景回复，不用每次手动提醒她。

**「小说模式」** —— 首页切到「世界」Tab，导入一本小说 TXT 文件，AI 会提取世界观，你可以选择身份进入书里的世界亲历故事。右侧实时显示你的修为、物品、NPC 关系、当前地点。

【图片位置：小说模式界面截图，右侧状态面板】

**「群聊」** —— 在角色设定里可以创建群聊，拉入2-8个角色，像真的群聊一样，你发一条消息，多个角色会轮流回复。可以@某个角色让她单独回应你。

## 一些常见问题

**Q：我的对话记录在哪里，安全吗？**
A：全部存在你自己浏览器的本地存储里（localStorage），关掉浏览器不会消失，换浏览器/换设备需要手动导出备份。网站本身没有服务器存你的数据。

**Q：可以多人用同一个 API Key 吗？**
A：技术上可以，但 API Key 就是你的消费账户，建议自己单独申请，消费有迹可查。

**Q：AI 角色能记住多久以前的对话吗？**
A：有「章节摘要」功能，旧对话会自动压缩成摘要存档，AI 能通过摘要了解很久以前的内容，不会完全忘记。

**Q：万一清理了浏览器缓存怎么办？**
A：在设置里可以把所有角色数据导出成 JSON 文件，下次导入回来就好了，建议定期备份。
